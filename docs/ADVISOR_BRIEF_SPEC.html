Advisor Brief — CSS Specification
Complete styling instructions for cowork. Every value here is final. Do not deviate without discussion. This document covers: design tokens, page structure, component-by-component CSS, typography rules, spacing, and the things that must never change.

Section 2
Typography — sizes, weights, rules
Never use: Inter, Roboto, or system fonts for UI text. DM Sans only. Never use Playfair Display for anything other than the page eyebrow, page headline, and card headlines.
Never abbreviate: Write "Money Market Fund" not "MMF". "After-tax" not "AT". "Operating Cash" not "Op Cash". Full names everywhere, always.
Numbers: All amounts use font-variant-numeric: tabular-nums and letter-spacing: -0.02em minimum. Weight 300 for display amounts, weight 500 for totals.

Section 3
Page structure — top to bottom
Page background
background: hsl(220,5%,93%) on body. This is the cool near-white that shows between and around all cards.
Page gutter
All page-level elements: margin: 0 32px or padding: 0 32px. Nothing bleeds to the edge.
Page header
Padding: 24px 32px 0. Background: var(--page). Contains: left side (eyebrow + headline + subtitle), right side (compose button + 1440/1920 toggle + date).
Compose button — position
Move to the page header, right side, above the 1440/1920 toggle. Remove the floating pill. It should be a navy outlined button: border 1px solid var(--navy), color var(--navy), background transparent, padding 7px 16px, font-size 11px weight 600. When clicked opens the compose flow.
Space below header
margin-bottom: 16px on the ph-top flex row before the situation overview
Situation overview
Full-width card, margin 0 32px. Two-column grid: 120px label left, 1fr body right. No top margin — connects directly to the header below.
Eyebrow stat bar
Flush below the situation overview. border-top: none so they read as one connected unit. Same left/right margin as situation overview.
Card grid
Padding: 16px 32px 32px. display: grid, grid-template-columns: repeat(3, 1fr), gap: 20px. Gap is 20px not 12px — cards need separation to feel like distinct executable actions.
Max card width
No explicit max-width — let the grid handle it. But the 32px gutter on each side plus 20px gaps means each card is ~(viewport − 64px − 40px) / 3. At 1440px that's ~445px which is correct.

Section 10
Hard rules — never deviate
Never: border-radius on any card, button, badge, input, or container. Everything is square-cornered.
Never: box-shadow on any element.
Never: More than one colored KPI number per card. The dominant number gets the card color. Everything else is plain navy.
Never: Abbreviate. "Money Market Fund" not "MMF". "Operating Cash" not "Op Cash". "After-tax" not "AT". Full names always.
Never: Inter, Roboto, or system-ui as the primary font. DM Sans only for UI. Playfair Display only for eyebrow, page headline, and card headlines.
Never: A floating compose button that overlaps content. Compose action belongs in the page header, top right.
Never: Two green dots / two status indicators in the page subtitle. One clean line: "GURU analysis complete · Ready to execute" in green, no dots.
Never: "50 bps expected rate cut" as a KPI in the interest rates card. The three KPIs are: rate-vulnerable cash (dominant, amber), after-tax income at risk, T-bill rate available.
